## User Summary

Introduces a plugin-shared shell-scripts library at `plugins/lwndev-sdlc/scripts/` containing ten deterministic utilities (ID allocation, title slugging, requirement-doc resolution, branch-name construction, branch create-or-switch, single-checkbox flipping, section-wide checkbox flipping, conventional-commit emission, PR creation, and branch-name classification). Every sibling SDLC skill replaces its prose recipe for the same operation with a one-line `bash ${CLAUDE_PLUGIN_ROOT}/scripts/<name>.sh` invocation, so identical logic stops being retyped across twelve SKILL.md files. Ships with a bats test fixture per script, a `pr-body.tmpl` asset consumed by `create-pr.sh`, a `README.md`, and a vitest integration test that asserts presence and executable bits. Version bump and CHANGELOG edits are deferred to `/releasing-plugins` and are out of scope.

## Non-applicable dimensions

- a11y: no UI surface. Scripts are CLI-only, emit to stdout/stderr, and have no screen-reader or keyboard-navigation concerns.
- i18n (user-facing strings): error messages are intentionally terse English tokens (e.g. `error: no file matches`) consumed by orchestrator logic, not end-users. Localization is out of scope; RTL / combining-character robustness for input is probed under Inputs and Environment above.
- Clock skew / daylight-saving / leap seconds: no script uses timestamps or time arithmetic. Timestamps in emitted artifacts are provided by the calling skill, not these scripts.
- Database / queue / multi-system dependency cascades: scripts depend only on `git`, `gh`, and (optionally) `jq`. No database, queue, or downstream service integration exists in this change.
- Authentication / authorization / under-privileged user: the only auth surface is `gh` (delegated to the user's existing gh credentials) and `git push` (delegated to the user's git credentials). These scripts do not introduce a new auth surface.
