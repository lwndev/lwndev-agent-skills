## User Summary

`prepare-fork.sh` is a new plugin-shared Bash script that the orchestrator calls once at every forked SDLC step to run the pre-fork ceremony in a single composed invocation. It takes a workflow ID, a zero-based step index, a canonical skill-name, and optional `--mode`/`--phase`/`--cli-model`/`--cli-complexity`/`--cli-model-for` flags; it verifies the sub-skill's `SKILL.md` is readable, resolves the FEAT-014 model tier via `workflow-state.sh resolve-tier`, appends a `modelSelections` audit-trail entry to the state file via `workflow-state.sh record-model-selection`, and emits an FR-14 echo line on stderr. The resolved tier is printed on stdout so the orchestrator captures it with `tier=$(bash prepare-fork.sh ...)` and passes it verbatim as the Agent tool's `model` parameter. The change replaces ~400–600 tokens of procedural prose at ~10 fork sites per feature workflow with a single scripted call.

## Non-applicable dimensions

- a11y: `prepare-fork.sh` is a non-interactive Bash script with no UI surface (no terminal UI, no colors, no prompts). Its only human-visible output is the FR-14 echo line which is plain ASCII (plus one em-dash U+2014 character in the Edge Case 11 warning variant, tested under Environment → non-UTF8 locale). There is no screen-reader, keyboard-navigation, or focus-trapping concern.
- i18n (beyond locale testing): The echo line prose is English-only by design — this is a developer/orchestrator-facing log, not an end-user message. The timestamp format is locale-independent. The tier labels (`haiku`, `sonnet`, `opus`) are product names that do not translate. Full i18n coverage is deferred until the orchestrator's user-facing summaries are localized (out of scope for FEAT-021).
